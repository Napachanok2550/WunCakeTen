## รายละเอียดโดยย่อ
โครงงานนี้เป็นการพัฒนาแอปพลิเคชันบนสมาร์ทโฟนสำหรับการทบทวนและเยียวยาจิตใจผ่านการตอบคำถามวันละ 1 คำถาม โดยคำถามจะถูกออกแบบมาเพื่อกระตุ้นให้ผู้ใช้ได้สำรวจความคิด ความรู้สึก และเหตุการณ์ที่พบเจอในชีวิตประจำวัน เปรียบเสมือนการเขียนบันทึกส่วนตัวในรูปแบบดิจิทัลที่เข้าถึงได้ง่ายและใช้งานได้ทุกวัน

วัตถุประสงค์ของโครงงาน
1. เพื่อพัฒนาแอปพลิเคชันบนสมาร์ทโฟนที่ช่วยให้ผู้ใช้สามารถทบทวนและถ่ายทอดความคิดความรู้สึกของตนเองผ่านการตอบคำถามประจำวัน
2. เพื่อช่วยลดความเครียดในชีวิตประจำวันของผู้ใช้ทุกเพศทุกวัย โดยเป็นพื้นที่ปลอดภัยในการระบายความรู้สึกอย่างเป็นส่วนตัว
3. เพื่อส่งเสริมการเข้าใจตนเองและการดูแลสุขภาพจิต

## คุณลักษณะและขอบเขต
คุณลักษณะ
1. ระบบแสดงคำถามประจำวันวันละ 1 คำถามให้ผู้ใช้
2. ผู้ใช้สามารถบันทึกคำตอบและเรียกดูข้อมูลย้อนหลังได้
3. แอปพลิเคชันใช้งานง่าย และข้อมูลเป็นส่วนตัวของผู้ใช้

ขอบเขต
1. เป็นแอปพลิเคชันบนสมาร์ทโฟนสำหรับตอบคำถามและบันทึกความรู้สึกประจำวัน
2. ระบบจัดเก็บข้อมูลภายในแอปพลิเคชันเพื่อใช้งานส่วนบุคคล
3. แอปพลิเคชันไม่รองรับการแชร์ข้อมูล และไม่ใช่โปรแกรมทางการแพทย์

### คุณลักษณะขั้นต่ำ
- เป็นแอปพลิเคชันสำหรับตอบคำถามและบันทึกความรู้สึกประจำวัน โดยข้อมูลประกอบด้วย วันที่ คำถาม และคำตอบของผู้ใช้
- ผู้ใช้สามารถสร้าง เพิ่ม แก้ไข และลบข้อมูลคำตอบของตนเองได้
- มีหน้าจอการใช้งานอย่างน้อย 2 หน้า ได้แก่ หน้าจอแสดงคำถามและบันทึกคำตอบประจำวัน และหน้าจอแสดงข้อมูลย้อนหลัง
- ระบบสามารถบันทึกข้อมูลคำถามและคำตอบของผู้ใช้ลงไฟล์ข้อมูล เช่น CSV หรือ JSON
- ผู้ใช้สามารถเปิดไฟล์ข้อมูลที่บันทึกไว้กลับมาแสดง แก้ไข และใช้งานในภายหลังได้

### คุณลักษณะเพิ่มเติม
- ผู้ใช้สามารถเลือกหมวดหมู่คำถาม เช่น ความคิด ความรู้สึก หรือเหตุการณ์ในชีวิตประจำวัน
- สามารถค้นหาและเรียกดูคำตอบย้อนหลังตามวันที่หรือคำสำคัญได้
- ผู้ใช้สามารถเพิ่มระดับอารมณ์หรือสัญลักษณ์แทนความรู้สึกในแต่ละวันได้
- ระบบสามารถแสดงสรุปการใช้งาน เช่น จำนวนวันที่ตอบคำถาม หรือความสม่ำเสมอในการใช้งาน
- สามารถตั้งค่าการแจ้งเตือนเพื่อเตือนให้ผู้ใช้เข้ามาตอบคำถามประจำวันได้

## แผนการดำเนินโครงงาน
| วางแผนและศึกษา | สัปดาห์ที่ 1 | ได้แนวคิดโครงงาน ขอบเขตงาน และศึกษาพื้นฐาน JavaFX พร้อมแหล่งอ้างอิง |
| ออกแบบระบบ | สัปดาห์ที่ 2 | ได้โครงสร้างโปรแกรม โครงสร้างข้อมูล และออกแบบหน้าจอการใช้งานคร่าวๆ |
| พัฒนาโปรแกรม | สัปดาห์ที่ 3-4 | โปรแกรมสามารถแสดงคำถาม เพิ่ม แก้ไข ลบ และบันทึกข้อมูลได้ |
| พัฒนาและทดสอบ | สัปดาห์ที่ 5-6 | โปรแกรมทำงานได้ครบตาม MVP มีอย่างน้อย 2 หน้าจอ และบันทึก/โหลดข้อมูลได้ |
| จัดทำเอกสารและปรับปรุง | สัปดาห์ที่ 7 | จัดทำให้เอกสารโครงงานและสไลด์นำเสนอมีความสมบูรณ์มากที่สุด |
| สรุปผลการดำเนินงาน | สัปดาห์ที่ 8 | ได้ข้อสรุปผลการพัฒนา ปัญหา อุปสรรค และสิ่งที่ได้เรียนรู้จากการทำโครงงาน |

**หมายเหตุ**
- แผนการดำเนินงานอาจมีการปรับเปลี่ยนตามความเหมาะสมของเวลาและความคืบหน้าในการพัฒนาโครงงาน
- หากพบปัญหาด้านเทคนิคหรือข้อจำกัดด้านเวลา จะให้ความสำคัญกับการพัฒนาในส่วนคุณลักษณะขั้นต่ำ(MVP) เป็นลำดับแรก
- สมาชิกในกลุ่มจะติดตามและประเมินความคืบหน้าของโครงงานร่วมกันอย่างสม่ำเสมอ

## การแบ่งงาน
นางสาวธัญญาภรณ์ มหารมย์ :
- ช่วยออกแบบภาพรวมของโปรแกรม
- จัดทำเอกสารโครงงาน
- เขียนส่วนอธิบายการทำงานของโปรแกรม และสรุปผลการดำเนินงาน
- รวบรวมผลงานจากสมาชิกทุกคนและตรวจสอบความเรียบร้อยของเอกสาร
นายธีรภัทร อายุเจริญกูล :
- ออกแบบหน้าจอการใช้งาน (UI)
- ช่วยพัฒนาโปรแกรมในส่วนของการแสดงข้อมูลย้อนหลัง การเพิ่ม แก้ไข และลบข้อมูล
- ทดสอบการใช้งานของโปรแกรม
- ช่วยจัดเตรียมภาพประกอบหรือ Prototype สำหรับใช้ในเอกสารโครงงาน
 นางสาวนภชนก รุ่งสกุลลิขิต :
- วางโครงสร้างและแนวคิดของแอปพลิเคชัน
- พัฒนาโปรแกรมด้วย JavaFX เช่น หน้าจอหลัก หน้าบันทึกคำตอบ และการเชื่อมโยงหน้าจอ
- ออกแบบโครงสร้างข้อมูล และจัดการการบันทึก–โหลดข้อมูล (CSV / JSON)
- ตรวจสอบความถูกต้อง และแก้ไขข้อผิดพลาดของโปรแกรม

## ความท้าทายและความเสี่ยง
1. ความท้าทายด้านเทคนิค
  - ยังไม่คุ้นเคยกับการพัฒนาแอปพลิเคชันด้วย JavaFX โดยเฉพาะการออกแบบหน้าจอและการเชื่อมโยงข้อมูลระหว่างหลายหน้าจอ
  - แนวทางแก้ไข: ศึกษาเอกสารและตัวอย่างจากเว็บไซต์ทางการของ JavaFX และบทเรียนออนไลน์ เช่น OpenJFX Documentation 
  และ JavaFX Tutorial บน YouTube
2. ความท้าทายด้านการจัดการข้อมูล
  - การบันทึกและเรียกใช้ข้อมูลคำถามและคำตอบจากไฟล์ (เช่น CSV หรือ JSON) อาจเกิดความผิดพลาดในการอ่านหรือเขียนข้อมูล
  - แนวทางแก้ไข: ทดลองพัฒนาฟังก์ชันบันทึกและโหลดข้อมูลตั้งแต่ระยะแรก และทดสอบกับข้อมูลตัวอย่างอย่างสม่ำเสมอ
3. ความเสี่ยงด้านเวลา
  - หากใช้เวลามากเกินไปกับการออกแบบส่วนติดต่อผู้ใช้หรือฟีเจอร์เพิ่มเติม
  อาจทำให้ไม่สามารถพัฒนาโครงงานให้เสร็จตามกำหนดเวลาได้
  - แนวทางแก้ไข: กำหนดขอบเขตของฟีเจอร์ขั้นต่ำ (MVP) ให้ชัดเจน และวางแผนการทำงานเป็นขั้นตอนพร้อมติดตามความคืบหน้าของโครงงานอย่างสม่ำเสมอ

## ต้นแบบและเอกสารอ้างอิง
โครงงานนี้ได้รับแรงบันดาลใจมาจากหนังสือ เกิดมาทำไม 365 คำถาม
ใช้แนวคิดการตั้งคำถามรายวันเพื่อทบทวนชีวิต ความคิด และความรู้สึก
(นำมาประยุกต์จากรูปแบบหนังสือ เป็นแอปพลิเคชันบนสมาร์ทโฟน)

แหล่งเรียนรู้และโค้ดอ้างอิง
- OpenJFX (JavaFX Documentation)
https://openjfx.io
ใช้ศึกษาโครงสร้าง JavaFX การสร้างหน้าจอ และการจัดการ UI
- JavaFX Tutorials (Oracle)
https://docs.oracle.com/javase/8/javase-clienttechnologies.htm
ใช้อ้างอิงแนวทางการพัฒนาแอปพลิเคชันด้วย JavaFX
- YouTube – JavaFX Tutorial
https://www.youtube.com/results?search_query=javafx+tutorial
ใช้เรียนรู้การสร้างหลายหน้าจอ (Multiple Scenes) และการจัดการ Event
- GitHub – JavaFX Example Projects
https://github.com/topics/javafx
ใช้เป็นตัวอย่างโครงสร้างโปรเจกต์และการจัดการไฟล์ข้อมูล
